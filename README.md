# CRM Сервісного Центру

Внутрішня CRM система для сервісного центру з ремонту електроприладів.

## Опис

Ця система дозволяє:
- Вести облік замовлень на ремонт
- Відстежувати статус ремонтів
- Управляти контактними даними клієнтів
- Вести облік доходів і витрат
- Керувати доступом користувачів до системи

## Технології

- Python 3.8+
- Flask
- SQLAlchemy
- PostgreSQL
- Flask-Login
- Bootstrap 5
- Chart.js

## Встановлення

1. Клонуйте репозиторій:
```
git clone <url_репозиторію>
cd <назва_папки>
```

2. Створіть віртуальне середовище та активуйте його:
```
python -m venv .venv
# Windows
.venv\Scripts\activate
# Linux/Mac
source .venv/bin/activate
```

3. Встановіть залежності:
```
pip install flask flask-login sqlalchemy psycopg2-binary werkzeug
```

4. Створіть базу даних PostgreSQL:
```
createdb service_crm
```

5. Налаштуйте з'єднання з базою даних у файлі app.py:
```python
app.config['SQLALCHEMY_DATABASE_URI'] = 'postgresql+psycopg2://postgres:ваш_пароль@localhost:5432/service_crm'
```

## Запуск

1. Ініціалізуйте базу даних (тільки перший раз):
```
# Запустіть сервер
python app.py

# Відкрийте в браузері
http://localhost:5000/init_db
```

2. Запустіть сервер:
```
python app.py
```

3. Відкрийте у браузері:
```
http://localhost:5000
```

## Початковий доступ

Після ініціалізації бази даних створюється користувач за замовчуванням:
- Логін: admin
- Пароль: admin

Рекомендується змінити пароль після першого входу через адміністративну панель.

## Структура проекту

- `app.py` - основний файл програми з маршрутами та логікою
- `models.py` - моделі бази даних
- `templates/` - HTML-шаблони
- `styles/` - CSS-файли

## Примітки щодо безпеки

Ця система призначена для внутрішнього використання і не містить усіх необхідних механізмів захисту для публічного розгортання. Перед використанням у виробничому середовищі рекомендується додати:

- HTTPS
- Більш надійну автентифікацію
- Захист від CSRF-атак
- Обмеження швидкості запитів
- Логування подій безпеки
